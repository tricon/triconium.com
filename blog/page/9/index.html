Super Magic Trackpad Turbo HD Remix

Nov 13th, 2010

[[posterous-content:9xnYZ5MgHp4QjNianBAJ]] I recently purchased Apple’s Magic Trackpad off of a friend (thanks, Jeff), and I have come to love it. Not only do I find trackpads more accurate and direct than mice, a third party tool has allowed me to replicate all of the functionality of my MX Revolution and more. The app: BetterTouchTool [[posterous-content:mb2FVhvJjKmeNznQ5NIJ]] BetterTouchTool allows you to remap the gestures of the touchpad to virtually anything you wish. Want to remap four finger swipe down? No problem. Pinch to zoom? Yep. Not only can you remap these gestures globally, you can remap them on a per-application basis. This allows you to create uniformity across your apps. Just as a three finger swipe left and right allows you to go back and forth between folders in Finder, the same gesture can be applied to iTunes to navigate tracks. [[posterous-content:9I3bc7VmrJHKSmnEsJSU]] BetterTouchTool also provides a number of gestures that aren’t directly supported by Apple: five finger gestures, tiptap gestures, complete thumb cancellation, and more. BetterTouchTool isn’t only limited to the Magic Trackpad, however. It can also be used with the Magic Mouse as well as the trackpad built in on your MacBook, providing it has multitouch support. In addition, it offers advanced controls and experimental features for perfecting your input methods. [[posterous-content:UTGEXGJdpus9E9exD8ho]] Do note: BetterTouchTool is still in development and far from complete. Some gestures don’t pick up every time which is frustrating. Regardless, it still proves useful for extending the capabilities of your multitouch peripheral and will only get better in time.

If you utilize multitouch peripherals, this is a no brainer. Download BetterTouchTool now.

An Apt Point on Languages and Tools by Mike C

Nov 9th, 2010

In response to this article, Mike C in the comments writes:

Mike C

In the phrase “I want a language that assumes everything I will be building is an MVC web app, and builds that right into the core language, not just a library” you’re missing a key ingredient: the IDE. For those of you who have never used MIcrosoft’s Visual Studio for writing an ASP.NET MVC application, the amount of work that the IDE does for you is incredible. While you could all of the files needed for a web app in Notepad (or gEdit, if you’re into that) the amount of code you would need to write is considerable — it’s precisely the kind of “boilerplate” crap that should be handled for you by the developing environment.

In addition to handling a lot of the coding work for you (but certainly not “magic coding”) it’s important to not overlook the ability to debug an application in the IDE. When writing .NET code I can walk line-by-line through a running application, looking at variable values. In PHP I have to do something like “echo $var; end;” and keep moving that “debug in the browser” echo statement until I find the problem.

I like the idea of having a language specifically created for the purpose of making a web application, but without a holistic view — web server/mod, web framework, language, IDE, storage/model extensibility, client-side technology/jQuery — this whole conversation is pretty much meaningless.

And the biggest point the author is missing: HTML5 is the next revolution that will radically change the way we write the server-side code of our web apps.

While I’ve grown up with general purpose editors such as VIM, Emacs, and TextMate, I have come to appreciate the heavy lifting and perspective an IDE can provide. Take Xcode for example. It is certainly not without its flaws, but the boilerplate code it generates, the easy debugging, the code sense, and its other workflow-improving features, make developing Objective-C apps considerably easier than in an editor such as TextMate. (I certainly do love TextMate, though, and wish I could somehow embed it within Xcode — best of both worlds.)

It is these productivity enhancements that have made me yearn for a quality, native IDE for Rails development. NetBeans is the best I’ve found in terms of UI, but it still suffers from non-native UI elements that are a pain to use as well as slow, clunkiness in general.

Choosing an IDE or editor is certainly a matter of taste, but once applications grow beyond a certain size, it seems IDEs really begin to show their value and almost become a necessity.

Where do you stand? IDE or editor?

The Art of Unitasking

Nov 7th, 2010
[[posterous-content:EJtedfbkaouFldigDcdr]]

It used to be that your operating system effectively only let you run a single application at a time. Since computers weren’t very fast, users would have to shut down an application and wait for the next application to launch before they could resume their work. This was not effective.

Eventually the evolution of hardware and software brought about the oh-so-coveted multitasking. Great! Now people could run dozens of applications and switch between them on a whim. Even better, users could view two applications side by side or watch a movie while they worked; because, you know, humans are incredibly efficient machines and can look in two directions at once.

Sarcasm aside, multitasking has enabled us to accomplish more while allowing us to accomplish less. The problem is that our brains only multitask on certain levels. Cognitively, our brains more-so Fast Context Switch, which is akin to the Fast Application Switching that occurs on mobile devices today. Just as there is overhead when computers switch contexts, there is overhead when your brain does so as well. Have you ever experienced a sudden lack of awareness, like you needed to get your bearings, when switching from one application to the next? Your brain has changed state and it must now process all the new information that it has aggregated.

Multitasking is effective when actions are orderly chained: you just finished a web design mockup in Photoshop so you are now switching to Fireworks to slice it, or you are switching to TextMate to implement it. Multitasking is devastating when switching between unrelated contexts: you’re working in TextMate and flip over to Safari to check Twitter. Ouch! You must now order and process information all over again once you switch back to TextMate.

Sometimes you can get your bearings quickly, but these context switches add up over time. It’s even worse when you were in deep thought, when you had concentration or were “wired in” as they say in “The Social Network”, and you now have to earn that all over again – and concentration is truly something you earn with great effort and practice.

Looking at iOS

iOS is a prime example of cognitive unitasking and proper multitasking. iOS allows only certain processes to be run in the background. On a technical level this saves on resources. iOS also only allows one application to take up the screen at any given time. On a cognitive level, this encourages unitasking; and iOS has the perfect animation for switching contexts.

Applications literally switch places. This gives an immediate visual cue and emotional feeling that contexts are being switched. It also gives the user a moment to prepare for the new context.

Compare this to the previous animation in iOS when opening a link in Safari from within another application (skip to 4:17).

The current application zooms away while Safari zooms in. This creates a completely different emotional feeling, one that is far more jarring and makes the user feel less in control.

Regardless, iOS effectively enforces unitasking. The result is devoted attention. Consequently, switching applications is easy yet painful in that it requires additional effort to do so – a good reason to have a workflow defined.

Looking at Lion

Apple is bringing the art of unitasking to the desktop and laptop OS in their upcoming release of OS X: Lion. While switching applications is as easy as ever, and more orderly thanks to the new Mission Control UI, Apple is encouraging applications to have a fullscreen mode, providing a native API for doing so. The result is this:

Easy. Organic.

Fullscreen modes allow the user interface to be reorganized in a completely different way, focusing the user on the given application and the tasks at hand. Fullscreen is undeniably powerful in its ability to encourage and enforce unitasking, and as desktops and laptops generally have larger screens than mobile devices, applications can still be laid out side by side for those moments when you need to reference one application while working within another.

Multitasking is Overrated

On a functional level, multitasking serves a very important purpose. On a higher level, it is adverse to productivity. Multitasking has become a buzzword in which many think it synonymous with productivity; but as Merlin Mann said in ”Broken Meetings”, multitasking is something everyone says they’re good at, but in practice, isn’t.

We’re seeing a trend towards a new hybrid of multitasking empowered unitasking – multitasking done right. Forgo multitasking. Embrace unitasking. Accomplish more with less.

P.S. Empower yourself. The ability to produce and deliver isn’t resolved by unitasking at a technical level. Learn to focus, become mindful, and discover new techniques.

Organizational ADD

Nov 6th, 2010

Over the years I have tried many different forms of digital organization. The search-don’t-sort method, where you throw things in generic folders such as Documents then search for them, proved too chaotic. It also discouraged browsing, which can remind one of forgotten documents as well as trigger new thoughts and concept

Meticulously sorting documents into folders is often too time consuming. While it is “browsing friendly”, it lacks flexibility. Does an article on historical fiction go under fiction, history, or writing?

Tagging can supplement both searching and sorting methods, but it can also be a burden and a headache. Do you tag both singular nouns and plural? What about synonyms? How much or little do you tag? These questions make tagging undesirable.

So if none of these methods are the answer, what is?

Enter DEVONthink

DEVONthink allows you to search, sort, and tag as much as desired, but more importantly, it creates relationships between your documents. It analyzes keywords, and over time, offers increasingly intelligent suggestions for related documents, as well as where to place the document in your organizational hierarchy. While it can’t analyze the contents of every file, it does accept any file into its database. This effectively turns DEVONthink into a super-Finder; and DEVONthink databases are Spotlight friendly.

An Example Workflow

[[posterous-content:vJcgCiCcnwszfblJesim]]

Use databases to separate your files. Separate databases are more portable than large, encompassing databases.

 

[[posterous-content:gsrrBpsEHDkIcnzenEEh]]

Use folders to separate contexts and smart folders to aggregate documents.


[[posterous-content:fHweGqhchFfGlBCgAoyw]]

Use tags to describe attributes that the analytics can not infer. An example would be describing departments or document statuses.


[[posterous-content:cAzFBhazryCGrkltykau]]

See Also and Classify propose related documents and organizational locations for the selected document.

Of course one can mix and match these methods to their liking, but having the power of analytics greatly saves on the time required to organize effectively.

 

What methods have you found work well for you? What is your ideal method for digital organization?

Copy as bit.ly for LaunchBar

Nov 3rd, 2010

Want to quickly make a short URL using bit.ly and use LaunchBar? The script below should help you.

Unzip it and place it in ~/Library/Scripts, open the file, and replace “USERNAME” and “APIKEY” with your bit.ly username and api key. If the script doesn’t show up in LaunchBar, you need to add the Scripts folder to your LaunchBar index. To do so:

  1. Activate LaunchBar
  2. Click the gear
  3. Go to Index –> Show Index
  4. Click the + in the bottom left
  5. Add the folder ~/Library/Scripts

From now on, any scripts you add to that folder will be indexed by LaunchBar.

To use the script:

  1. Activate LaunchBar
  2. Begin typing part of the name of the script
  3. Hit space
  4. Paste your URL
  5. Hit enter Your shortened URL will automatically be copied to the clipboard!

Download the script here.
