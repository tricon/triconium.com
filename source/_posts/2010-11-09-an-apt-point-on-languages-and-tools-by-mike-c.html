In response to this article, Mike C in the comments writes:

Mike C

In the phrase “I want a language that assumes everything I will be building is an MVC web app, and builds that right into the core language, not just a library” you’re missing a key ingredient: the IDE. For those of you who have never used MIcrosoft’s Visual Studio for writing an ASP.NET MVC application, the amount of work that the IDE does for you is incredible. While you could all of the files needed for a web app in Notepad (or gEdit, if you’re into that) the amount of code you would need to write is considerable — it’s precisely the kind of “boilerplate” crap that should be handled for you by the developing environment.

In addition to handling a lot of the coding work for you (but certainly not “magic coding”) it’s important to not overlook the ability to debug an application in the IDE. When writing .NET code I can walk line-by-line through a running application, looking at variable values. In PHP I have to do something like “echo $var; end;” and keep moving that “debug in the browser” echo statement until I find the problem.

I like the idea of having a language specifically created for the purpose of making a web application, but without a holistic view — web server/mod, web framework, language, IDE, storage/model extensibility, client-side technology/jQuery — this whole conversation is pretty much meaningless.

And the biggest point the author is missing: HTML5 is the next revolution that will radically change the way we write the server-side code of our web apps.

While I’ve grown up with general purpose editors such as VIM, Emacs, and TextMate, I have come to appreciate the heavy lifting and perspective an IDE can provide. Take Xcode for example. It is certainly not without its flaws, but the boilerplate code it generates, the easy debugging, the code sense, and its other workflow-improving features, make developing Objective-C apps considerably easier than in an editor such as TextMate. (I certainly do love TextMate, though, and wish I could somehow embed it within Xcode — best of both worlds.)

It is these productivity enhancements that have made me yearn for a quality, native IDE for Rails development. NetBeans is the best I’ve found in terms of UI, but it still suffers from non-native UI elements that are a pain to use as well as slow, clunkiness in general.

Choosing an IDE or editor is certainly a matter of taste, but once applications grow beyond a certain size, it seems IDEs really begin to show their value and almost become a necessity.

Where do you stand? IDE or editor?
